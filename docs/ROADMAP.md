# ROADMAP.md — Космічна ферма

Розробка ізометричної гри про виживання на мертвій планеті. Гравець вирощує рослини в куполі, які генерують кисень. Керування — мишка (кліки по об'єктам).

---

## 1. Цілі MVP

**Що має мати гра на кінець 3 тижнів:**
- ізометрична карта з керуванням мишкою;
- посадка, ріст, збір рослин (3 типи);
- система кисню та виживання;
- інвентар, ракетний термінал, базова економіка;
- збереження та завантаження гри (save/load).

---

## 2. Базовий баланс

### Рослини
| Рослина | Час росту | Кисень | Ціна | Примітка |
|---------|-----------|--------|------|----------|
| Швидка біомаса | 30 сек | +2% | 50 $ | Швидка |
| Кисневий аероб | 120 сек | +8% | 100 $ | Ефективна |
| Кристалічний томат | 180 сек | +3% | 300 $ | Дорога |

### Кисень
- **Витрата поза куполом:** 2% за 10 сек;
- **Генерація від рослин:** див. таблицю;
- **Критичний рівень:** < 20% (червоне попередження);
- **Смерть:** 0% кисню.

### Апгрейди
| Апгрейд | Ціна | Ефект |
|---------|------|-------|
| +1 грядка | 200 $ | Нова грядка |
| Кисневий пілон | 500 $ | +15% кисню |
| Кращий скафандр | 300 $ | +50% часу в космосі |

### Інвентар
- Простий список: ресурс + кількість;
- Максимум 20 одиниць на вид;
- **Save/load:** зберігає інвентар, рівень кисню, стан рослин, гроші.

---

## 3. План на 3 тижні

### **Тиждень 1 — База гри**

**Завдання:** Java + LibGDX, ізометрична карта, камера, mouse input, звуки UI, базові анімації.

**Розподіл по ролях:**
- **Lead Developer:** проєкт, Entity/Grid/Camera, mouse input logic, save/load API, звуковий менеджер;
- **Gameplay Programmer:** entity структури, логіка обробки подій, базова система анімацій;
- **UI/UX Developer:** mouse events, hover-стани, перший UI, звуки на клік;
- **Game Designer & Tech Artist:** карта Tiled (.tmx), спрайти тайлів, звуки (клік, наведення), простші fade-in анімації.

**Ассети:** .tmx карта, спрайти тайлів, звуки UI (2-3 шт).

**Контрольна точка:** ✅ Гра запускається, карта видно, кліки мишкою визначають клітинку, звуки на UI, просте демо.

---

### **Тиждень 2 — Фермерство**

**Завдання:** грядки, посадка, ріст (3 стадії), збір, інвентар, UI вікна, анімації рослин, звуки гейм-подій.

**Розподіл по ролях:**
- **Lead Developer:** Plant/Plot/Inventory класи, save/load частина 1, інтеграція, система анімацій спрайтів;
- **Gameplay Programmer:** логіка росту, таймери, посадка, збір, управління рослинами, тригери анімацій;
- **UI/UX Developer:** вікна посадки, вибір рослини, інформація, інвентар UI, звуки на дії;
- **Game Designer & Tech Artist:** спрайти рослин (3 стадії × 3), спрайти грядок, баланс часу росту, анімації переходів стадій (fade/scale), звуки (посадка, ріст, збір).

**Ассети:** спрайти рослин, спрайти грядок, дизайн вікон, звуки (посадка, ріст, збір — 3 шт).

**Контрольна точка:** ✅ Посадка → ріст (видно анімацію) → збір цикл працює, звуки на кожну дію, інвентар збирає врожай, гравець розуміє UI.

---

### **Тиждень 3 — Кисень, економіка, MVP**

**Завдання:** OxygenManager, HUD, ракетний термінал, апгрейди, save/load завершення, анімації HUD, звуки критичних подій, фінальна збірка.

**Розподіл по ролях:**
- **Lead Developer:** OxygenManager, зони (під куполом/космос), save/load частина 2, інтеграція економіки, система анімацій UI;
- **Gameplay Programmer:** логіка кисню (витрата/генерація), скафандр (таймер), смерть, терміналу (продаж) логіка;
- **UI/UX Developer:** HUD (кисень, таймер, попередження) з анімаціями, вікно терміналу, апгрейди, обробка всіх кліків, звуки інтерактивні;
- **Game Designer & Tech Artist:** баланс кисню (витрата/генерація), ціни, вартість апгрейдів, спрайти UI, анімації (pulse попередження, flash апгрейду), звуки (смерть, продаж, апгрейд — 3 шт), тестування, баг-фікси.

**Ассети:** спрайти HUD, дизайн вікна терміналу, анімації UI-елементів, звуки (смерть, продаж, апгрейд).

**Контрольна точка:** ✅ HUD показує кисень з анімацією, вихід за межи купола = витрата скафандра, рослини генерують кисень, звуки на дії, продаж ресурсів, апгрейди з анімацією, цикл посадка→ріст→продаж→апгрейд, демо готове.

---

## 4. Залежності та синхронізація

**Передача між тижнями:**
- Т1 → Т2: Entity struct, Mouse Input API, Save/Load API;
- Т2 → Т3: Plant класи, Inventory, UI framework.

**Щоденна стендап (5 хв):**
- Що закінчено, на чому працюють, блокери.

**Тестування (кінець тижня):**
- Вся команда грає, перевіряє критерії.
- Критичні баги — одразу, менші — в наступний тиждень.

---

## 5. Резервний план

- **Т1:** гра запускається, кліки работают, базові звуки UI;
- **Т2:** посадка + ріст + збір (без інвентарю); анімації рослин + звуки основних дій;
- **Т3:** кисень + простий продаж (без апгрейдів); HUD із базовою анімацією.

**Опціональні фішки** (якщо встигаєте раніше): дрони, додаткові рослини, музика, складніші анімації переходів.

---

## 6. Ризики та рішення

| Ризик | Рішення |
|-------|---------|
| Забагато фіч | Тримати MVP, не розширюватись раніше часу |
| Складна ізометрія | Спочатку просто: клік → дія |
| Save/load зламається | Окремо тестувати, мокувати спочатку |
| Баланс кривий | Короткі плейтести після кожного тижня |
| Баги з mouse input | Логування, правильні хітбокси |

---

## 7. Критерії готовності MVP

✅ Запускається; 

✅ Клік → посадка;

✅ Рост через таймер;

✅ Збір → інвентар;

✅ HUD кисень;

✅ Космос → таймер скафандра;

✅ Ракетний термінал продає;

✅ Апгрейди працюють;

✅ Save/load;


---

## Підсумок

**Core-loop:** клік → посадка → ріст → продаж → апгрейд → більше ресурсів.

Якщо цей цикл працює, гру можна нарощувати без великих ризиків.
